# Domain Documentation

This project uses a **single-context** layout for domain documentation.

## Locations

- **Context**: `conductor/` (root) - Describes the domain language and core concepts.
- **Architecture Decisions**: `conductor/tracks/` `conductor/archive` - Records architectural decisions (ADRs).

## Consumer Rules

- Skills should read `conductor/product.md` to understand business logic and terminology.
- Skills should check `conductor/tracks/` before proposing significant architectural changes.
